**Reason for supersession:**
The old tag was signed against `tscp-v0.6.1-dogfood`, which predates the
`tscp-protocol` crate extraction that landed on `master`. It uses the
old `crates/prover-server/src/proof_envelope.rs` layout and
`prover_server::proof_envelope` imports, which no longer match the
current codebase.

The current tag reflects work done directly on top of current `master`:
- `tscp-protocol`'s `Cargo.toml` fixed to build as `rlib` + `cdylib`
  (it was `cdylib`-only, which silently broke the crate's own tests)
- `ProofEnvelope::open()` now takes an explicit `verifier_version`
  argument instead of assuming the verifier is always V0.6.1
- Full 4-case version-boundary conformance matrix
  (0.6.1/0.6.2 envelope × 0.6.1/0.6.2 verifier)
- `prove_handler` now calls `sumcheck_verify` on its own output before
  responding, closing a soundness gap where proofs were shipped
  unverified

The old tag is left in place as an immutable historical artifact and
is not deleted or rewritten.
